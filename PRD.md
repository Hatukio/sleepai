# 催眠 App MVP - 产品需求文档（PRD）

**版本：** v1.0
**创建日期：** 2026-03-17
**产品名称：** SleepAI（暂定）

---

## 1. 产品概述

### 1.1 产品定位
一个基于 AI 的个性化催眠故事生成器，通过分析用户的职业、压力源和偏好场景，生成定制化的催眠音频，帮助失眠用户快速入睡。

### 1.2 目标用户
- **主要用户：** 25-45 岁的失眠人群
- **细分群体：**
  - 工作压力大（程序员、教师、销售等）
  - 生活焦虑（青年、职场新人）
  - 偶尔失眠（需要助眠工具的人）

### 1.3 核心价值
- **个性化：** 每个用户的催眠故事都是独一无二的
- **快速：** 5 分钟内生成，无需等待
- **有效：** 专业催眠脚本 + 高质量 TTS
- **简单：** 5 个问题，一键生成

## 2. 功能需求

### 2.1 核心功能（MVP）

#### 功能 1：用户问卷
**优先级：** P0（必须有）
**页面：** `/`

**需求描述：**
用户访问首页后，填写 5 个问题，系统收集信息用于生成个性化催眠故事。

**问题设计：**

| 问题 | 类型 | 选项 | 说明 |
|------|------|------|------|
| Q1 | 单选输入 | 文本框 | "你是做什么工作的？" |
| Q2 | 多选 | 工作压力、情感问题、健康焦虑、经济压力、其他 | "最近让你压力最大的是什么？"（可多选） |
| Q3 | 单选 | 宁静的森林、温柔的海浪、浩瀚的星空、童年的老家 | "你喜欢什么场景？" |
| Q4 | 单选 | 温柔的女声、沉稳的男声 | "你喜欢什么样的声音？" |
| Q5 | 单选 | 5分钟快速入睡、10分钟标准时长、20分钟深度放松 | "你希望在多久内入睡？" |

**交互设计：**
- 5 个问题单页展示，减少操作步骤
- 提交按钮："生成我的催眠故事"
- 输入验证：所有问题必填

**数据存储：**
- 问卷答案存入数据库（脱敏处理）
- 不收集用户个人信息（姓名、手机号等）

---

#### 功能 2：催眠脚本生成
**优先级：** P0（必须有）
**后端：** `/api/generate-story`

**需求描述：**
根据用户问卷答案，调用 GPT-4 API 生成定制化的催眠脚本。

**Prompt 模板：**

```
你是一位专业的催眠治疗师，请为以下用户定制一个催眠故事。

用户画像：
- 职业：{occupation}
- 压力源：{stressors}（用中文逗号分隔）
- 喜欢的场景：{scene}
- 期望时长：{duration}分钟

要求：
1. 开头（30秒）：温和引导用户放松，调整呼吸节奏
2. 中间（大部分时长）：构建用户喜欢的场景，引导进入潜意识
3. 结尾（30秒）：温和暗示入睡，逐渐淡化声音
4. 语言风格：{voice_style}（温柔女声用温暖柔和的语气，沉稳男声用平稳安定的语气）
5. 字数控制在 {word_count} 字左右（按每分钟 100 字估算）

场景风格说明：
- 宁静的森林：描述鸟鸣、微风、树叶沙沙声、阳光透过树叶的光影
- 温柔的海浪：描述海浪拍岸、海风、沙滩、远处的海鸥
- 浩瀚的星空：描述星星闪烁、银河、宁静的夜空、宇宙的浩瀚
- 童年的老家：描述老房子的温暖、童年的回忆、熟悉的声音、安全感

压力源应对：
- 工作压力：强调放下工作、放松紧绷的神经
- 情感问题：引导释放情绪、拥抱内心平静
- 健康焦虑：强调身体放松、免疫系统强大
- 经济压力：引导放下忧虑、享受当下
- 其他：强调整体放松、释放所有压力

请直接输出催眠故事文本，不要包含标题、引言或其他说明文字。
```

**参数映射：**
- `word_count`：5 分钟 → 500 字，10 分钟 → 1000 字，20 分钟 → 2000 字
- `voice_style`：温柔的女声 → "温柔柔和"，沉稳的男声 → "平稳安定"

**返回格式：**
```json
{
  "success": true,
  "script": "生成的催眠脚本文本"
}
```

**错误处理：**
- GPT-4 API 调用失败 → 返回错误信息
- 生成内容异常 → 使用预设的默认脚本作为备选

#### 功能 3：音频生成
**优先级：** P0（必须有）
**后端：** `/api/generate-audio`

**需求描述：**
将生成的催眠脚本通过 TTS 转换为音频文件。

**调用方式：**
使用现有的 `coze-voice-gen` skill 的 TTS 功能

**参数：**
- `text`：催眠脚本
- `voice`：根据用户选择
  - 温柔的女声 → 使用温柔女声模型
  - 沉稳的男声 → 使用沉稳男声模型

**流程：**
1. 调用 coze-voice-gen 生成音频
2. 接收音频文件（MP3 格式）
3. 上传到云存储（七牛云/阿里云 OSS）
4. 返回音频访问 URL

**返回格式：**
```json
{
  "success": true,
  "audioUrl": "https://storage.example.com/audio/xxx.mp3",
  "duration": 600  // 音频时长（秒）
}
```

**备用方案：**
如果 coze-voice-gen 不可用，使用其他 TTS API（如 Azure TTS、阿里云 TTS）

#### 功能 5：反馈收集
**优先级：** P1（重要）
**页面：** `/feedback?storyId={storyId}`

**需求描述：**
用户听完音频后，引导用户填写反馈问卷，收集使用体验和付费意愿。

**反馈问题：**

| 问题 | 类型 | 选项 |
|------|------|------|
| Q1 | 星级评分 | 1-5 星（这个催眠故事对你有帮助吗？） |
| Q2 | 单选 | 完整听完、听了一半、听了几分钟就关了 |
| Q3 | 文本框 | 你最喜欢/不喜欢的地方是什么？ |
| Q4 | 单选 | 不愿意付费、5元/月、15元/月、30元/月、50元/月或更多 |
| Q5 | 文本框 | 你还想要什么功能？ |

**数据存储：**
- 反馈数据存入数据库
- 关联到对应的催眠故事 ID

**激励方式（可选）：**
- 填写反馈后，获得"后续免费会员"权益
- 或者小额奖励（如 5 元红包）

## 3. 非功能需求

### 3.1 性能要求
- 问卷提交响应时间 < 1 秒
- 催眠脚本生成时间 < 30 秒
- 音频生成时间 < 2 分钟
- 音频加载时间 < 3 秒

### 3.2 可用性要求
- 支持 PC 和移动端浏览器
- 支持主流浏览器（Chrome、Safari、Edge）
- 界面简洁直观，无需说明书

### 3.3 数据安全要求
- 问卷答案脱敏存储
- 不收集用户个人信息
- 音频文件加密存储（可选）
- API 调用使用环境变量，不暴露密钥

### 3.4 可扩展性要求
- 数据库设计预留扩展空间
- API 接口设计支持未来功能扩展

## 4. 数据模型

### 4.1 用户（User）

```prisma
model User {
  id        String   @id @default(uuid())
  createdAt DateTime @default(now())
  updatedAt DateTime @updatedAt

  stories   HypnosisStory[]
  usages    UsageRecord[]
}
```

### 4.2 催眠故事（HypnosisStory）

```prisma
model HypnosisStory {
  id          String   @id @default(uuid())
  userId      String
  user        User     @relation(fields: [userId], references: [id])

  occupation  String
  stressors   String   // JSON 数组字符串
  scene       String
  voiceStyle  String
  duration    Int      // 期望时长（分钟）

  script      String   // 生成的脚本
  audioUrl    String   // 音频 URL
  audioLength Int      // 音频实际时长（秒）

  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  usages      UsageRecord[]
  feedbacks   Feedback[]
}

model UsageRecord {
  id          String   @id @default(uuid())
  storyId     String
  story       HypnosisStory @relation(fields: [storyId], references: [id])

  startTime   DateTime
  endTime     DateTime?
  completed   Boolean  @default(false)
  createdAt   DateTime @default(now())
}

model Feedback {
  id          String   @id @default(uuid())
  storyId     String
  story       HypnosisStory @relation(fields: [storyId], references: [id])

  rating      Int      // 1-5
  completed   String   // 完整听完/听了一半/听了几分钟就关了
  likes       String?
  dislikes   String?
  willingness String   // 不愿意付费/5元/月/15元/月/30元/月/50元/月或更多
  moreFeatures String?
  createdAt   DateTime @default(now())
}
```

## 6. UI/UX 设计要点

### 6.1 设计原则
- **极简：** 界面简洁，减少干扰
- **温暖：** 色调柔和，营造放松氛围

- 主色：深蓝色（#1e3a5f）- 宁静、夜晚
- 辅助色：淡紫色（#9b72aa）- 温柔、放松
- 背景色：深灰色（#0f172a）- 沉稳、舒适
- 文字色：浅灰色（#e2e8f0）- 柔和、不刺眼

### 6.3 字体
- 标题：思源黑体（加粗）
- 正文：思源宋体（柔和）
