# 催眠 App MVP - 2周冲刺计划

## 🎯 项目目标

在 2 周内开发并测试一个 AI 定制化催眠故事生成器的 MVP，验证市场需求和用户付费意愿。

---

## 📊 核心指标

**成功标准：**
- 至少 100 人完成测试
- 使用完成率 ≥ 30%（完整听完整个音频）
- 付费意愿 ≥ 30%
- 收到至少 50 条有效反馈

## 🛠️ 技术架构

### 前端
```
技术栈：Next.js 14 + Tailwind CSS + shadcn/ui

页面结构：
├── / (首页 - 问卷入口)
├── /generate (生成页面 - 显示问题)
├── /player (播放页面 - 音频播放器)
└── /feedback (反馈页面 - 问卷)
```

### 后端
```
技术栈：Next.js API Routes + Prisma ORM

数据模型：
├── User (用户信息)
├── HypnosisStory (催眠故事)
└── UsageRecord (使用记录)

API 端点：
├── POST /api/generate-story (生成催眠脚本)
├── POST /api/generate-audio (生成音频)
├── GET /api/stories/:id (获取故事)
└── POST /api/feedback (提交反馈)
```

### AI 服务
```
- GPT-4 API：生成催眠脚本
- coze-voice-gen：TTS 音频生成
```

### 存储
```
- PostgreSQL：用户数据、故事内容
- 云存储（七牛/阿里云）：音频文件
```

## 📱 核心功能

### 1. 用户问卷（5 个问题）

```
Q1: 你是做什么工作的？
  [输入框]

Q2: 最近让你压力最大的是什么？
  [多选] 工作压力 | 情感问题 | 健康焦虑 | 经济压力 | 其他

Q3: 你喜欢什么场景？
  [单选] 宁静的森林 | 温柔的海浪 | 浩瀚的星空 | 童年的老家

Q4: 你喜欢什么样的声音？
  [单选] 温柔的女声 | 沉稳的男声

Q5: 你希望在多久内入睡？
  [单选] 5分钟快速入睡 | 10分钟标准时长 | 20分钟深度放松
```

### 2. AI 催眠脚本生成

**Prompt 模板：**
```
你是一位专业的催眠治疗师，请为以下用户定制一个催眠故事。

用户画像：
- 职业：{occupation}
- 压力源：{stressors}
- 喜欢的场景：{scene}
- 期望时长：{duration}分钟

要求：
1. 开头：温和引导用户放松，呼吸调整
2. 中间：构建用户喜欢的场景，引导进入潜意识
3. 结尾：温和暗示入睡，逐渐淡化声音
4. 语言风格：{voice_style}（温柔/沉稳）
5. 字数控制在 {word_count} 字左右

请直接输出催眠故事文本，不要包含其他说明。
```

### 3. 音频生成

**流程：**
```
1. 调用 coze-voice-gen 的 TTS 功能
2. 输入：生成的催眠脚本
3. 输出：音频文件（MP3）
4. 上传到云存储，获取访问 URL
```

### 4. 音频播放器

**功能：**
- 播放/暂停
- 进度条
- 音量控制
- 循环播放
- 下载功能

## 🚦 风险控制

### 可能遇到的问题
1. **生成质量不稳定** → 准备多个 Prompt 模板，A/B 测试
2. **TTS 效果不好** → 准备备用方案（其他 TTS 服务）
3. **用户测试招募困难** → 扩大推广渠道，提供小奖励（如后续免费会员）
4. **服务器不稳定** → 使用可靠的云服务商，做好监控
5. **负面反馈过多** → 快速调整，优先修复核心问题

### 止损点
如果以下情况发生，考虑止损：
- Day 7 无法完成基本功能
- Day 10 用户招募不足 30 人
- Day 12 满意度 < 2.5/5.0
- Day 13 付费意愿 < 10%
